## Bug 01 - SEC-01: IDOR xem tien do hoc tap
- Severity: Critical
- Nguyen nhan: `/api/v1/progress/[childUid]` va `/api/v1/course-run/[childUid]` khong xac thuc bearer token, cho phep doi UID tren URL.
- Cach sua: them `requireChildAccess` dung Firebase Admin verify token; kid chi doc/ghi UID cua minh, parent chi doc child co `linkedParentUid`, admin bypass. Ap dung them cho `/api/v1/progress/lesson-completion`.
- Tieu chi pass: Kid A goi progress/course-run/lesson-completion cua Kid B tra `403`; parent xem con cua minh tra `200`.

## Bug 02 - ERR-03: Huy Pro bi cat quyen ngay
- Severity: Medium
- Nguyen nhan: webhook `customer.subscription.deleted` ha `plan` xuong Free ngay, khong ton trong `current_period_end`.
- Cach sua: luu `currentPeriodEnd`, `expiresAt`, `cancelAtPeriodEnd`; neu huy cuoi ky va chua qua han thi giu `plan: pro`, `status: active` den het ky.
- Tieu chi pass: cancel-at-period-end van giu Pro den ngay het han; sau het han moi Free.

## Bug 03 - Pro khong dong bo Admin dashboard
- Severity: High
- Nguyen nhan: checkout chi update `subscriptions`, mot so UI/admin profile can doc trang thai tu `users`.
- Cach sua: webhook checkout/admin subscription update ca `subscriptions/{parentUid}` va mirror `users/{parentUid}` voi `isPro`, `plan`, `subscriptionStatus`, `proExpiresAt`.
- Tieu chi pass: mua Pro xong Admin Users hien PRO VIP va API subscription tra Pro.

## Bug 04 - SEC-02: Backdoor tu nang admin
- Severity: Critical
- Nguyen nhan: `AdminGuard` co Dev Helper goi client Firestore update `role: admin`; rules cu cho owner write toan bo `users/{uid}`.
- Cach sua: xoa hoan toan Dev Helper; user khong phai admin chi thay man chan quyen. Firestore rules chan owner update cac field nhay cam `role`, `isPro`, `plan`, `linkedParentUid`, `childUids`.
- Tieu chi pass: user thuong vao `/admin` khong co nut promote; client Firestore update `role` bi deny.

## Bug 05 - ERR-04: Thieu chan file upload >20MB
